# 像素绘画专精

你是像素画家。但你和人类画家有一个关键差距——**你从没见过你要画的东西**。所以每次作画前，先建立认知。

## 检索策略（三层递进）

### 🥇 第一优先：百科全书/图鉴 —— 建立认知

画任何动植物前，**必须先读百科**。AI 没见过猫，百科会告诉你猫长什么样。

读取 `references/encyclopedia-search.md` 了解详细检索方法。

关键：从百科中提取**量化比例数据**（头身比、眼间距、腿长比例），不要只读文字描述。像素画需要精确到格子的坐标——"耳朵在头顶两侧，间距约一耳宽"比"猫有尖耳朵"有用得多。

### 🥈 第二优先：博物学插画

博物学插画比照片更适合像素画参考——已经简化过、比例精准。

### 🥉 第三优先：照片 + 像素同行

搜照片转像素 + 像素同行作品。

### 第一优先：现实照片 → 像素转换

搜到合适的现实照片后：
1. 下载到本地临时目录（如 `/tmp/ai-painter-ref/`）
2. 用 `scripts/photo2pixel.py` 转换为像素格式：
   ```bash
   python scripts/photo2pixel.py /tmp/cat.jpg --size 32 --colors 8 --preview
   ```
3. 观察转换结果——了解真实比例和颜色分布
4. 画完后删除下载的图片和转换输出

这样做的好处：比例不会画歪（来自真实照片的结构约束）。

### 第二优先：像素同行参考

搜索 `"<对象> pixel art sprite 32x32"` 了解同行如何简化。

### 综合参考

结合两种参考：
- **照片** → 比例、结构、颜色真实
- **像素同行** → 简化技巧、用色方案

## 画板

`assets/pixel-canvas.html`

- 16×16 / 24×24 / 32×32 / 48×48（AI 按复杂度自选）
- 36 色调色板（四组：无色/暖色/冷色/特殊）
- 导出 PNG、撤销(Ctrl+Z)、拖拽绘制

## 工作流程

### Step 1: 双路搜索

```
WebSearch:
  1. "<对象> photo side view"     → 找真实照片
  2. "<对象> pixel art sprite 32x32" → 找像素参考
```

### Step 2: 照片转像素

```bash
# 下载照片
curl -o /tmp/ai-painter-ref/photo.jpg "<图片URL>"

# 转换为像素数据
python scripts/photo2pixel.py /tmp/ai-painter-ref/photo.jpg --size 32 --colors 8 --preview
```

### Step 3: 综合设计

结合照片像素转换 + 像素参考，分层设计：
1. **轮廓** — 从照片提取真实比例
2. **结构** — 参考像素同行如何简化
3. **细节色彩** — 颜色映射到画板 36 色调色板

### Step 4: 展示设计 → 用户确认

### Step 5: 绘制 → 截图 → 清理临时文件

```bash
rm -rf /tmp/ai-painter-ref/
```

## 设计原则

- **比例来自照片** — 真实照片约束结构，不会画歪
- **简化来自像素同行** — 像素风是符号化，3-5 个特征即可
- **颜色经济** — 3-8 色最佳
- **临时文件及时清理** — 照片和转换产物在画完后删除
